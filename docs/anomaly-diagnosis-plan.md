# 出租车异常行程诊断平台：当前实现与下一阶段路线图

## 当前已实现能力

### 单条行程异常诊断

- 已提供独立诊断页面 `/anomaly`，保留原有 `/trip` 和 `/car` 页面不变。
- 已支持 5 类规则型异常检测：
  - 绕路
  - 异常停留
  - 速度突变
  - 定位漂移
  - 异常跳点
- 已输出解释型诊断结果，包括：
  - 风险等级 `low / medium / high`
  - 100 分制异常评分 `anomaly_score`
  - 异常事件列表 `events`
  - 诊断指标 `metrics`
  - 地图高亮分段 `segments`
- 诊断结果按请求实时计算，当前未做结果落库。

### 地图回放与诊断展示

- 页面已支持轨迹地图回放、播放/暂停/重置、倍速控制。
- 已支持异常点和异常段高亮展示，并按异常类型使用固定颜色：
  - 绕路 `#f59e0b`
  - 停留 `#ef4444`
  - 速度突变 `#8b5cf6`
  - 漂移 `#06b6d4`
  - 跳点 `#f97316`
- 已支持点击异常列表后联动地图定位，并可跳转到对应异常片段。
- 已提供风险卡片、异常详情、触发证据和两类统计图：
  - 异常类型分布
  - 速度剖面与异常位置

### 异常路段分布

- 已提供异常路段分布接口和前端展示，用于识别最近一批行程中的高频异常道路。
- 当前路段分布基于诊断结果中的异常分段按 `road_id` 聚合，输出：
  - 道路编号
  - 异常命中次数
  - 涉及行程数
  - 主导异常类型
  - 最高异常级别
  - 平均速度
  - 样例行程
- 前端已支持“异常路段分布”地图和路段列表联动，并可从路段项跳转到样例行程诊断页。

### 异常车辆排行

- 已提供异常车辆排行接口和前端展示，不再属于“V1 未实现”范围。
- 当前排行基于最近一批行程做轻量实时聚合，输出：
  - 车辆编号
  - 采样行程数
  - 异常事件总数
  - 高风险行程数
  - 平均异常评分
  - 最差行程
  - 主导异常类型
- 前端已支持从排行项跳转到车辆画像页和最差行程诊断页。

### 当前测试覆盖

- 后端已提供规则测试，覆盖以下场景：
  - 正常轨迹无异常
  - 长时间停留
  - 速度突变
  - 异常跳点
  - 定位漂移
  - 绕路

## 当前对外接口与页面结构

### 后端接口

- `GET /api/trips/{trip_id}/diagnosis`
  - 用于单条行程异常诊断。
  - 核心响应结构：
    - `trip`
    - `summary`
    - `metrics`
    - `events`
    - `segments`
- `GET /api/anomaly/vehicles`
  - 用于异常车辆排行。
  - 当前支持参数：
    - `limit`
    - `trip_sample`
    - `per_vehicle`
  - 核心响应结构：
    - `sample_trip_count`
    - `vehicle_count`
    - `items`
- `GET /api/anomaly/roads`
  - 用于异常路段分布。
  - 当前支持参数：
    - `limit`
    - `trip_sample`
  - 核心响应结构：
    - `sample_trip_count`
    - `road_count`
    - `items`

### 关键数据结构

- `TripDiagnosisResponse`
  - `trip`: 行程基础信息与轨迹点
  - `summary`: 风险等级、评分、事件总数、各类型计数
  - `metrics`: 直线距离、实际距离、绕行系数、最高速度、停留总时长、重复道路占比、回摆次数
  - `events`: 诊断事件列表
  - `segments`: 地图渲染分段
- `AnomalyVehicleRankingResponse`
  - `sample_trip_count`: 当前排行统计使用的样本行程数
  - `vehicle_count`: 命中异常的车辆数
  - `items`: 排行项列表
- `AnomalyRoadDistributionResponse`
  - `sample_trip_count`: 当前路段分布统计使用的样本行程数
  - `road_count`: 命中异常的道路数
  - `items`: 路段分布项列表

### 前端页面结构

- `/anomaly`
  - 查询区：输入 `trip_id` 发起诊断
  - 地图区：轨迹回放、异常点/段标注、跳转定位
  - 结果区：风险卡片、异常事件列表、异常详情
  - 统计区：异常分布图、速度剖面图
  - 聚合区：异常路段分布、异常车辆排行
- `/trip`
  - 行程基础分析和拥堵分段回放
- `/car`
  - 车辆画像、两小时统计和车辆行程列表

## 当前边界与已知缺口

- 诊断结果尚未持久化，刷新或重复请求会重新计算。
- 阈值仍写死在后端规则常量中，前端没有参数面板。
- 车辆排行目前是轻量实时聚合，不是完整的离线批处理任务体系。
- 异常路段分布目前基于代表性分段和道路编号聚合，尚未扩展到网格热力、道路拓扑或更细粒度空间聚类。
- 当前排行样本来自最近一批行程，结果更适合演示和分析入口，不适合作为长期稳定运营指标。
- 当前测试重点在规则正确性，尚未形成完整的接口自动化与前端交互测试体系。

## 下一阶段路线图

### 1. 异常车辆排行增强

- 目标能力：
  - 提升排行的可解释性、可筛选性和可追踪性。
  - 支持更稳定的样本口径，而不仅是“最近 N 条行程”。
- 需要新增或扩展的接口：
  - 在 `/api/anomaly/vehicles` 上补充筛选能力，如时间范围、最小异常数、异常类型过滤。
  - 增加单车异常摘要接口，返回该车在指定范围内的异常趋势和代表性行程。
- 页面展示方式：
  - 在排行区域增加筛选栏、类型筛选、风险排序切换和趋势摘要。
  - 支持查看某车辆的异常构成占比和最近异常行程列表。
- 与当前 V1 的关系：
  - 当前版本已具备基础排行入口，这一阶段是在现有排行之上做增强，而不是推翻重做。

### 2. 异常路段分布增强

- 目标能力：
  - 从当前的道路聚合列表扩展到更完整的空间热点分析，识别高频异常区域和连续异常道路。
- 需要新增或扩展的接口：
  - 在现有 `/api/anomaly/roads` 上增加异常类型、时间范围、最小命中次数等筛选参数。
  - 增加面向网格热力或空间聚类的聚合接口，返回地图可直接渲染的热力数据。
- 页面展示方式：
  - 在现有路段分布地图基础上增加热力层、类型切换和空间过滤。
  - 支持按异常类型切换查看，例如只看跳点高发路段或停留高发区域。
- 与当前 V1 的关系：
  - 当前版本已具备基础道路聚合能力，这一阶段重点是把“道路列表”升级成“空间分析视图”。

### 3. 批量聚合与统计任务

- 目标能力：
  - 建立面向多车辆、多行程的批量分析链路，降低每次请求实时扫描的成本。
- 需要新增或扩展的接口：
  - 新增批量分析任务接口或内部调度入口，用于按时间窗口生成聚合结果。
  - 新增任务状态或统计结果查询接口，供前端和后续服务复用。
- 页面展示方式：
  - 前端可以显示批量统计时间窗口、统计口径和最近一次更新状态。
- 与当前 V1 的关系：
  - 当前排行仍是实时聚合，这一阶段是把“轻量实时”升级为“稳定批量统计”。

### 4. 结果持久化与查询优化

- 目标能力：
  - 将诊断结果和聚合结果持久化，提升查询速度并为历史分析提供基础。
- 需要新增或扩展的接口：
  - 新增诊断结果读取接口或缓存命中逻辑。
  - 为排行和路段分析提供基于持久化结果的查询接口。
- 页面展示方式：
  - 支持展示最近分析时间、缓存命中状态和历史结果回看。
- 与当前 V1 的关系：
  - 当前所有诊断均按请求实时计算，这一阶段重点解决性能和历史可追溯问题。

### 5. 阈值配置与精度优化

- 目标能力：
  - 在保留规则法可解释性的前提下，提升规则可调性并减少误判。
- 需要新增或扩展的接口：
  - 为诊断接口增加可选阈值参数，或增加规则配置读取接口。
  - 补充规则版本号和参数说明，方便结果解释。
- 页面展示方式：
  - 在诊断页增加参数面板或高级设置区，支持演示不同阈值下的诊断结果。
  - 对异常事件补充更细粒度的解释文本和证据说明。
- 与当前 V1 的关系：
  - 当前规则阈值固定在后端常量中，这一阶段是在现有规则框架内做配置化和精度提升。
